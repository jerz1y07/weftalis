# Open Workflow Registry Website

This directory contains the static Next.js website for Open Workflow Registry. The website reads public Workflow metadata generated from validated Packages. It does not use a database, API route, upload form, or account system.

## Data source

The root `registry/registry.json` file is the source of truth for website data. Registry Builder scans local Packages, asks the Validator to validate them, and writes public metadata for accepted Packages. It does not copy complete Workflow source exports into the Registry.

From the repository root, generate the Registry with:

```bash
cd tools/registry-builder
npm run build-registry
```

Registry Builder never executes a Workflow.

## Sync Registry data

After Registry Builder finishes, copy the public Registry metadata into the website:

```bash
cd website
npm run sync-registry
npm run check-registry
```

The sync command checks the basic Registry structure and writes `generated/registry.json`. The check command verifies the generated file, required website fields, Workflow counts, unique IDs, and supported validation status values.

`generated/registry.json` is automatically generated and committed to Git so the website can be built independently later. Do not edit this file by hand. Update Packages, run Registry Builder, and sync again instead.

The Next.js pages import the generated JSON through `lib/registry.ts`. Data is read while the static site is built, not from the filesystem on every page request. The browser never receives complete Workflow source files.

## Local development

Install dependencies once if needed, then start the development server:

```bash
npm install
npm run dev
```

Open [http://localhost:3000](http://localhost:3000).

## Build the static website

Run the data checks and build:

```bash
npm run sync-registry
npm run check-registry
npm run lint
npm run build
```

Next.js writes the static export to `out/`. Dynamic Workflow detail paths are generated from Registry Workflow IDs at build time.

## Safety boundary

This website is a read-only catalog. It does not execute, install, publish, or connect Workflows. A `valid` status means the Package passed the current Validator rules; it does not guarantee absolute safety, correctness, quality, or suitability. Users must inspect every Package and apply human judgment before reuse.
