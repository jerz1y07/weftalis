# Weftalis v0.1.0 Release Checklist

This checklist separates release-candidate verification from the later tag and GitHub Release actions. Do not complete the final section without explicit maintainer approval.

## Candidate contents

- [x] `CHANGELOG.md` and the v0.1.0 release notes are accurate and reviewed.
- [x] README status, commands, architecture, limitations, and public URL are current.
- [x] Specification, Validator, Registry Builder, Registry, website, contribution, CI, and security claims match implemented behavior.
- [x] The two accepted Registry entries and zero rejected entries are intentional.

## Safety and repository audit

- [x] No private email, real secret, credential, API key, local absolute path, production `localhost`, or unintended placeholder domain is present.
- [x] Weftalis naming and capitalization are consistent; obsolete names appear only in explicit brand prohibitions, if at all.
- [x] Markdown links, GitHub YAML, and JSON parse successfully.
- [x] Generated Registry files and website Registry data are synchronized.
- [x] No unexpected large file or uncommitted change is unexplained.
- [x] `git diff --check` passes.

## Reproducibility checks

- [x] Validator: safe install, typecheck, tests, and build pass.
- [x] Registry Builder: safe install, typecheck, tests, build, `build-registry`, and `verify-registry` pass.
- [x] Website: safe install, sync, consistency check, lint, normal static build, and `/weftalis` Pages build pass.
- [x] Production: all six routes, `robots.txt`, and `sitemap.xml` return HTTP 200.
- [x] Production canonical URLs and Registry content match the published repository data, with no duplicated `/weftalis/weftalis` path.

## Final release actions — maintainer approval required

- [ ] Review and approve the complete release-candidate diff.
- [ ] Commit the approved candidate and confirm CI and Pages checks for that exact commit.
- [ ] Replace the changelog candidate label with the release date if needed.
- [ ] Create the annotated `v0.1.0` tag on the approved commit.
- [ ] Push the commit and tag.
- [ ] Publish the GitHub Release using `docs/releases/v0.1.0.md` after removing its draft notice.
