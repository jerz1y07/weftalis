# Weftalis Brand Guidelines

## 1. Official name

The official product name is **Weftalis**.

## 2. Official tagline

The open registry for reusable and verifiable workflows.

## 3. Short product description

Weftalis is an open-source registry for publishing, inspecting, validating, versioning, and reusing workflows across platforms.

## 4. Correct capitalization

Always write the user-facing product name as **Weftalis**. Use the lowercase form `weftalis` only when a technical identifier requires lowercase characters.

Do not use `WEFTALIS`, `wefTalis`, `WeftAlis`, `Flowgistry`, or `Trellis` as the product name.

## 5. Naming examples

- Weftalis
- Weftalis Validator
- Weftalis Registry Builder
- Weftalis Workflow Package Specification v0.1
- Weftalis Contributors
- `weftalis-validator`
- `weftalis-registry-builder`

The generic phrase “open workflow registry” may describe the product category, but it is not a replacement for the Weftalis brand name.

## 6. Terms that should not be renamed

Brand changes must not rename or redefine established technical identifiers and user content, including:

- `workflow.yaml` and `workflow.schema.json`
- `spec_version` and existing Schema field names
- Workflow Package IDs and Package folder names
- Registry JSON field names and existing Registry Entry data
- runtime platform IDs, including `n8n` and `dify`
- existing Workflow names
- website routes, CLI arguments, and Validator exit codes
- the current local repository folder and Git history

## 7. Current visual direction

The current visual direction favors:

- interconnected structures;
- woven paths;
- modular workflow nodes;
- open infrastructure;
- trust without claiming absolute safety; and
- a clean developer-tool aesthetic.

This direction does not require a complex logo, external font, image, or other external brand resource.
